# 素材 02：我与 AI 的四阶段关系演化

## 素材主题

第一章节《我与 AI》的开场主视觉。

这张图不是讲具体截图，也不是复述项目经历，而是把这些经历抽象成一条关系演化线：

> 从“模型入口”，到“模型拓展”，再到“Agent 出世”，最后到“掌控 Agent”。

关键修正：

- `向量化检索` 和 `GitHub AI 审核` 归入“模型拓展”，但它们是两个故事：一个拓展 AI 的知识面，一个拓展 AI 的应用场景。
- Agent 阶段不是从 Manus 才开始，而是先有 AutoGPT 式长任务研究：分任务、执行、验收、再执行，让 AI 尝试长时间工作。
- 这个阶段早期还没有今天成熟的 tool / tool calling 产品概念，更像“任务循环”和“自我推进”的范式探索。
- 2024.03 的 Cursor / vibe 页面、以及 2024 vibe 出来的 `alading` 项目，是用户已经广泛使用 Agent 的代表素材。
- Manus 在 2025 年把 Agent 概念带火，但不是用户第一次接触 Agent。

截图素材用于证明这些阶段真实发生过；这张图负责把它们抽象成一个可被听众快速理解的框架。

## 画面目标

参考用户给的 `Phase 1 / Phase 2 / Phase 3` 图，但改成 4 个阶段。

画面应该表现成：

- 横向 4 栏阶段图
- 每栏有一个简洁图形隐喻
- 每栏有短标题和一句解释
- 阶段之间用细线分隔或轻箭头连接
- 风格统一为电子墨水 + vivo 蓝强调色

画面不应该表现成：

- 时间轴流水账
- 个人项目截图拼贴
- 复杂产品架构图
- 机器人、人脑、握手等俗套 AI 视觉
- 过度科技发布会风格

## 四个阶段文案

### Phase 1：Model as Entry

中文标题：

> 模型入口

解释：

> ChatGPT 让模型进入大众视野，API 让它成为可接入产品的能力。

图形隐喻：

- 一个用户圆点
- 一个 `ChatGPT` 入口节点
- 一个 `API` 连接节点
- 一个页面 / Chat 窗口
- 一个模型节点
- 用一条短路径表达 `ChatGPT → API → 个人网页`

对应素材：

- `2023.5.11接入AI.PNG`

### Phase 2：Model Extension

中文标题：

> 模型拓展

解释：

> 模型没有变成 Agent 之前，先被外部系统拓展成更可用的能力。

调研补充：

- `知识拓展`：RAG / embedding / 向量检索，把外部知识接到模型上下文里。
- `行为拓展`：prompt、结构化输出、fine-tuning，让模型更稳定地按某种格式或风格工作。
- `能力拓展`：function calling / tool calling，让模型连接外部数据和动作。
- `场景拓展`：把模型放进搜索、客服、代码审核、办公流等真实业务场景。

本阶段分讲时只展开两个与你素材强相关的故事：

- `知识面拓展`：向量化检索拓展 AI 知识面。
- `应用场景拓展`：GitHub AI 审核拓展 AI 应用场景。

图形隐喻：

- 总图：模型节点位于中心，向外分出知识、行为、能力、场景四条支线。
- 分图一：知识点阵 / 向量检索路径，表达“知识面拓展”。
- 分图二：代码分支 / 审核标记，表达“应用场景拓展”。
- 所有支线都回连到模型节点，但不要画成 Agent 执行。

对应素材：

- `解除向量化检索拓展AI知识面.jpg`
- `2023.7.15github接入AI审核.JPG`

### Phase 3：Agent Emergence

中文标题：

> Agent 出世

解释：

> 从长任务循环到 vibe coding，AI 开始承担连续执行。

图形隐喻：

- 一个循环结构：分任务 → 执行 → 验收 → 再执行
- 一个 Agent 节点沿循环推进，旁边有页面 / 代码结果
- 末端有用户验证节点
- 表达“AI 不只是回答，而是开始长时间执行”

对应素材：

- `IMG_8023.PNG`
- 2024 Cursor / vibe coding 页面
- 2024 vibe 出来的 `alading` GitHub repo 截图
- 早期开源 / 公司内部 coding agent 使用截图

### Phase 4：Agent Control

中文标题：

> 掌控 Agent

解释：

> 借用 2x2 Prioritization Matrix，用重要程度和执行复杂度选择 Agent 掌控方式。

图形隐喻：

- 一个四象限坐标图
- 横轴：执行复杂度，左低右高
- 纵轴：重要程度，底部低，顶部高
- 左上象限：Quick Win，高重要、低复杂，轻量交付、结果验收
- 右上象限：重点项目，高重要、高复杂，拆小任务、多 Agent 并行、持续验收
- 左下象限：低优先填充，低重要、低复杂，自动化或顺手处理
- 右下象限：先搁置 / 先拆解，低重要、高复杂，不直接交给 Agent 狂奔
- 用户三个场景作为案例点贴到矩阵中，而不是直接作为象限名称

对应素材：

- Codex 阶段
- 公司级多 repo / 多 Agent 并行素材
- 开源项目 review / 架构 review / code review / BDD 行为测试素材
- 给别人做项目 / spec-kit 式开发 / 最终结果素材

## 语言描述版

一张横向 16:9 杂志风信息图，画面分成四个等宽栏目，从左到右依次是：`模型入口`、`模型拓展`、`Agent 出世`、`掌控 Agent`。每一栏顶部有 `Phase 1` 到 `Phase 4` 的小号等宽标记，中间是简洁线性图标，底部是一句短解释。

第一栏表现 `ChatGPT → API → 个人网页` 的短路径，强调“模型从公共入口变成个人产品能力”。第二栏表现模型节点向外分出四条拓展支线：`知识`、`行为`、`能力`、`场景`，其中知识和场景两条用 vivo 蓝强调，作为后面两张证据页的入口。第三栏表现 Agent 的早期范式：`分任务 → 执行 → 验收 → 再执行` 的循环，以及 2024 vibe coding 的页面 / repo 结果，强调“AI 开始长时间执行”。第四栏表现一个 2x2 Prioritization Matrix，横轴是执行复杂度，纵轴是重要程度：左上是 Quick Win，右上是重点项目，左下是低优先填充，右下是先拆解；再把公司级多 repo、开源维护、简单交付作为三个案例点贴进去，强调“掌控 Agent”。

整体使用电子墨水风格：纸张底色、黑白灰、细线条、轻网格、克制留白。只用少量 vivo 蓝 `#415fff` 强调阶段编号、连接线和关键 Agent 节点。不要出现真实截图，不要出现机器人、握手、巨大 AI 大脑。视觉像内部分享里的概念框架图，不像商业发布会海报。
