# 素材 03：我与 AI 四阶段分讲 Prompts

## 使用约定

- 总图用于第一章节《我与 AI》的开场：抽象关系演化，不放真实截图。
- 四个阶段图用于分讲：左侧是概念线稿，右侧预留素材图窗口，后期把真实截图塞进去。
- 统一使用 `guizang-ppt-skill` 的“电子杂志 × 电子墨水”方向。
- 统一强调色：vivo 蓝 `#415fff`。
- 所有图片默认 16:9 横向构图。
- 生成图里不要伪造产品截图、假 logo 或复杂 UI；截图位置只做空白占位框。

## 阶段边界

这里的关键不是年份，而是“AI 在协作关系中的职责变化”。

- `模型入口`：ChatGPT 让模型进入大众视野，API 让它成为可接入产品的能力。
- `模型拓展`：同一阶段模块里有两条故事线，一条拓展 AI 的知识面，一条拓展 AI 的应用场景。
- `Agent 出世`：AutoGPT 式长任务循环先出现，2024 我已经在 Cursor / vibe coding 中广泛使用 Agent，2025 Manus 才把 Agent 概念带火。
- `掌控 Agent`：借用 2x2 Prioritization Matrix，用重要程度和执行复杂度选择不同的 Agent 掌控方式。

注意：`向量化检索` 和 `GitHub AI 审核` 都放在第二阶段，但不要写成同一个动作。前者是拓展 AI 的知识面，后者是拓展 AI 的应用场景。

第二阶段在 PPT 里建议拆成 3 页：

- `02-0 模型拓展总图`：讲 AI 模型拓展的总体结构，不放个人截图。
- `02-1 知识面拓展`：讲 embedding / 向量化检索，后期放你的截图。
- `02-2 应用场景拓展`：讲 GitHub AI 审核，后期放你的截图。

第四阶段在 PPT 里建议拆成 4 页：

- `04-0 掌控 Agent 总图`：2x2 Prioritization Matrix，横轴执行复杂度，纵轴重要程度。
- `04-1 公司级重点项目`：高重要、高复杂，多 repo / 多 Agent 并行，小任务验收。
- `04-2 开源维护项目`：作为案例点，用架构 review、code review、BDD 行为测试掌控质量。
- `04-3 简单交付项目`：低复杂、短期交付，spec-kit 式开发，不看代码，只验收最终结果。

## 阶段二调研补充

阶段二不是只有你的两个场景，而是大模型从“可调用”走向“可用”的一组工程化拓展方式。

可收敛成四类：

- `知识拓展`：RAG、embedding、向量检索，把外部知识放进模型上下文。
- `行为拓展`：prompt、结构化输出、fine-tuning，让模型输出更稳定、更贴合任务。
- `能力拓展`：function calling / tool calling，让模型连接外部系统、数据和动作。
- `场景拓展`：把模型嵌入搜索、客服、代码审核、办公流等真实业务流程。

调研链接：

- RAG 论文：<https://papers.neurips.cc/paper/2020/hash/6b493230205f780e1bc26945df7481e5-Abstract.html>
- OpenAI Embeddings：<https://developers.openai.com/api/docs/guides/embeddings>
- OpenAI Function Calling：<https://platform.openai.com/docs/guides/function-calling>
- OpenAI Supervised Fine-tuning：<https://platform.openai.com/docs/guides/supervised-fine-tuning>
