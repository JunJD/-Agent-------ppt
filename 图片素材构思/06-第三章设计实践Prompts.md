# 素材 06：第三章设计实践 Prompts

## 使用约定

- 用于第三章《设计实践：回到我们自己的项目》。
- 这一章不展开底层实现细节，重点讲清楚 Agent 进入协同文档之后，如何被看见、被判断、被撤回、被管理。
- 统一使用 `guizang-ppt-skill` 的“电子杂志 × 电子墨水”方向。
- 统一强调色：vivo 蓝 `#415fff`。
- 所有图片默认 16:9 横向构图。
- 图中文字使用中文，标签短而清晰。
- 不生成假产品 UI、不生成假 logo、不画机器人、不画巨大 AI 大脑、不画握手隐喻。
- 这一章尽量避开底层字段名和实现细节，画面围绕 `协同文档`、`side-by-side diff`、`Ctrl+Z vs 撤回 Agent 修改`、`记录 / 展示 / 决策` 展开。

## 章节视觉主线

第三章不是在讲 Agent 的模型能力，而是在讲：

> 当 Agent 进入协同文档工作流，系统如何把它接进一个能看见、能判断、能回滚的闭环。

建议视觉顺序：

- `00 回到我们的项目`：章节转场，先把视角落回自己的项目。
- `01 Agent 以什么身份进入文档`：说明 Agent 附属于当前用户，但改动必须有独立归属边界。
- `02 side-by-side diff`：让用户先看懂修改，再决定接不接。
- `03 Ctrl+Z vs 撤回 Agent 修改`：把人的编辑和 Agent 的编辑分开。
- `04 记录 / 展示 / 决策`：把协同文档里的控制逻辑收成三层。
- `05 透明、可对比、可回滚`：章节收束，强调闭环能力。

用途：3.1 页，章节转场，回到自己的协同文档项目。

```text
生成一张横向杂志风章节幕封，表达：前面讲完 Coding Agent 和业务实践之后，第三章回到我们自己的协同文档项目。画面不要像发布会海报，而要像一张安静的工作台转场页：左侧是章节标题“回到我们的项目”，右侧是一个被抽象成工作区的协同文档环境，里面能看到协同文档、修改对比、撤回边界、决策按钮这几个节点，形成一条清晰但克制的工作流。整体气质要像从前两章的宏观讨论，慢慢落到一个真实项目里。电子墨水风格，黑白灰为主，少量 vivo 蓝强调色 #415fff，纸张质感、细线条、轻网格、编号、短标签、留白充足。图中文字使用中文，标签只保留：回到我们的项目、协同文档、对比、撤回、决策。不要真实截图、卡通、3D、霓虹科技感、机器人、巨大 AI 大脑、握手隐喻、SaaS 模板感或假 logo。输出必须是16:9横向构图，主体居中但保留边距，画面密度中等。只保留核心信息图本身，不要生成页眉、页脚、标题、页码、角标、署名、装饰边框、超长条、竖图或不规则比例。
```

## 放进 PPT 时的页面文案建议

- 回到我们的项目
- Agent 以什么身份进入文档
- 让用户先看懂修改
- `Ctrl+Z` 和撤回 Agent 不能混
- 记录、展示、决策三层闭环
- 透明、可对比、可回滚
