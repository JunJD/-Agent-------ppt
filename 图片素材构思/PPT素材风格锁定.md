# PPT 素材风格锁定

适用项目：《当 Agent 进入业务系统》

分享场景：内部分享。  
内容气质：个人观察 + 工程判断 + 产品设计思考。  
表达目标：让听众理解 Agent 进入业务系统的边界、控制点和落地方式，而不是做商业发布会式包装。

## Skill 模板现状

当前 `guizang-ppt-skill` 里不是多套互斥 PPT 模板，而是：

- 1 个 HTML 底层模板：`assets/template.html`
- 10 个页面布局骨架：封面、章节页、数据页、左文右图、图片网格、Pipeline、问题页、大引用、对比页、图文混排
- 5 套主题色预设：墨水经典、靛蓝瓷、森林墨、牛皮纸、沙丘
- 7 类配图提示词：人文纪实、杂志风信息图、Pipeline、对比图、系统关系图、UI 情景图、数据大字报

## 本项目锁定选择

### 1. HTML 模板

只使用：

- `assets/template.html`

不新增第二套 HTML 模板。

### 2. 主题色

锁定：

- vivo 风格强调色：`#415fff`

原因：

- 主题是 AI / Agent / 业务系统，属于内部技术与产品分享
- `#415fff` 有明确的 vivo 品牌识别，也足够适合 AI / Agent 主题
- 画面底色仍以电子墨水的黑白灰、纸张质感和留白为主
- `#415fff` 只作为少量强调色，用在线条、节点、编号、关键路径和少量标签上
- 适合内部分享里的“理性、克制、有判断”，同时保持品牌一致

不用：

- skill 预设主题色：本项目优先统一到 vivo 风格强调色
- 大面积纯蓝背景：会削弱电子墨水和杂志感
- 多种蓝紫渐变：容易滑向通用科技发布会风格

### 3. 页面布局

页面布局不是风格分叉，不删。  
它们是同一套视觉系统里的不同版式。

本项目优先使用：

- Layout 1：开场封面
- Layout 2：章节幕封
- Layout 4：左文右图
- Layout 6：两列流水线
- Layout 8：大引用页
- Layout 9：并列对比
- Layout 10：图文混排

谨慎使用：

- Layout 3：数据大字报，只有出现强数据时使用
- Layout 5：图片网格，只有有多张真实截图或对比素材时使用
- Layout 7：问题页，适合章节转折或结尾，不频繁使用

### 4. 配图提示词

所有概念类素材统一锁定为：

- 类型 2：杂志风信息图

原因：

- 最贴合“电子杂志 × 电子墨水”
- 适合表达 Agent、流程、边界、checkpoint、rollback 等抽象概念
- 比人物照片、机器人插画、复杂系统图更稳定
- 更适合内部分享：不煽动、不浮夸，强调结构理解

只在明确场景下使用其他类型：

- 类型 3 Pipeline 图：当页面只讲流程步骤
- 类型 4 对比图：当页面明确是旧模式 vs 新模式
- 类型 6 UI 情景图：当页面需要呈现协同办公笔记系统界面
- 类型 7 数据大字报：当页面只有一个核心数字

默认不用：

- 类型 1 人文纪实照片：容易和电子墨水信息图风格断裂
- 类型 5 系统关系图：除非页面明确讲模块架构，否则容易过于工程图

## 本项目统一配图 Prompt 母版

```text
生成一张横向杂志风信息图，解释：[页面概念]。电子墨水风格，黑白灰为主，少量 vivo 蓝强调色 #415fff，细线条、网格、编号、短标签、留白充足。图中文字使用中文，标签必须短而清晰。不要卡通、3D、霓虹科技感、机器人、巨大 AI 大脑、握手隐喻、SaaS 模板感或假 logo。输出必须是16:9横向构图，主体居中但保留边距，画面密度中等。只保留核心信息图本身，不要生成页眉、页脚、标题、页码、角标、署名、装饰边框、超长条、竖图或不规则比例。
```

## 使用规则

- 每个素材 MD 只保留一个首选 prompt
- 不保留“备选 prompt”，避免后续生成风格漂移
- 不为同一概念同时写照片版、科技插画版、系统图版
- 图片统一 16:9，除非已经确定页面落位不是主视觉
- 图片中文字统一中文
- 标签数量尽量控制在 5 个以内
- 不画具象机器人，不画握手，不画巨大 AI 大脑
- 内部分享场景下，画面优先表达结构和判断，不追求发布会式冲击力
- `#415fff` 只做强调，不做大面积铺底
